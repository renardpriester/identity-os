# IdentityOS Examples

## Purpose

This folder contains sample data used to demonstrate how IdentityOS processes identity lifecycle events, policy decisions, governance requirements, and audit outcomes.

The example files are not connected to real users or real organizations. They are fictional datasets designed to model enterprise identity scenarios.

---

## Current Example Files

| File                                                         | Purpose                                                         |
| ------------------------------------------------------------ | --------------------------------------------------------------- |
| [sample-identity-events.json](sample-identity-events.json)   | Contains sample identity lifecycle and governance events.       |
| [sample-policy-decisions.json](sample-policy-decisions.json) | Contains sample policy decisions linked to the identity events. |
| [sample-dashboard-metrics.json](sample-dashboard-metrics.json) | Contains sample dashboard metrics for executive identity risk, lifecycle operations, governance, risk scoring, access drift, and automation health. |

---

## Sample Identity Events

The `sample-identity-events.json` file contains example events that IdentityOS may receive from HR, ticketing, privileged access, governance, or identity systems.

Current sample event types include:

* Joiner
* Mover
* Leaver
* Contractor onboarding
* Contractor expiration
* Privileged access request
* Non-human identity review

These events represent the types of identity changes that require policy evaluation, access decisions, governance controls, and audit evidence.

---

## Sample Policy Decisions

The `sample-policy-decisions.json` file contains example decisions generated by the IdentityOS Policy Engine.

Each policy decision is linked to a related identity event.

A policy decision may include:

* Decision ID
* Related event ID
* Policy result
* Decision summary
* Access to grant
* Access to remove
* Controls required
* Governance requirements
* Risk level
* Review frequency
* Expiration requirement
* Audit reason

## Sample Dashboard Metrics

The `sample-dashboard-metrics.json` file contains sample metrics used to demonstrate IdentityOS dashboard concepts.

The dataset includes metrics for:

* Executive identity risk
* Lifecycle operations
* Governance
* Risk scoring
* Access drift
* Automation health

These metrics help demonstrate how IdentityOS can turn identity activity, policy decisions, risk scoring, access drift detection, and generated reports into dashboard-ready visibility.

The sample dashboard metrics support the v0.4 dashboard concept layer.


## Example Event-to-Decision Pattern

IdentityOS follows this basic pattern:

```text
Identity Event
      ↓
Policy Evaluation
      ↓
Policy Decision
      ↓
Access Action
      ↓
Governance Control
      ↓
Audit Evidence
```

Example:

```text
Joiner Event
      ↓
Evaluate department, job title, worker type, and role package
      ↓
Approve baseline access
      ↓
Grant Microsoft 365, Teams, and required business applications
      ↓
Apply MFA and Conditional Access
      ↓
Generate audit evidence
```

## Why Sample Data Matters

Sample data makes the architecture easier to understand.

It shows how IdentityOS could work in a real enterprise environment without requiring a live production tenant, real employees, or sensitive data.

These examples help demonstrate:

* How lifecycle events are structured
* How policy decisions are linked to events
* How governance requirements are applied
* How audit evidence can be produced
* How automation scripts can process identity data

---

## How the Examples Are Used

The current automation script reads these files:

```powershell
python automation/policy-evaluator.py
```

The script loads the sample identity events, matches them to policy decisions, and prints a readable evaluation summary.
