# IdentityOS Reports

## Purpose

This folder contains generated sample reports for IdentityOS.

Reports demonstrate how identity events, policy decisions, governance requirements, access actions, and risk indicators can be transformed into readable evidence for security, IAM, governance, audit, and leadership audiences.

---

## Current Reports

| Report                                                  | Purpose                                                                                                                  |
| ------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------ |
| [Sample Governance Report](sample-governance-report.md) | Summarizes sample policy decisions, governance requirements, risk levels, access remediation actions, and audit reasons. |

---

## Report Source

The current sample governance report is generated by:

```text
automation/generate-governance-report.py
```

The script reads data from:

```text
examples/sample-identity-events.json
examples/sample-policy-decisions.json
```

Then it produces:

```text
reports/sample-governance-report.md
```

## Why Reports Matter

Identity programs need more than workflows.

They need evidence.

Reports help show:

* What access was granted
* What access was removed
* Which decisions required approval
* Which identities were high risk
* Which identities were critical risk
* Which decisions required review
* Which governance controls were applied
* Why access decisions were made
* What audit evidence exists

This turns identity operations into measurable governance.
